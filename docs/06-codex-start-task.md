# Codex 开发启动任务

## 任务目标

在不重新设计、不自行生图的前提下，实现《上帝掷骰子吗？》v1.0 的可运行前端。

## 技术栈

- React
- TypeScript
- Vite
- CSS Modules 或普通 CSS
- 可选：Framer Motion
- `localStorage`
- `html2canvas` 或 `satori` 用于分享图

## 开发原则

1. 以仓库中的 PRD、文字库、视觉文档和交接文档为唯一需求源；
2. 不修改已确认文案；
3. 不新增社区、登录、注册、思维图谱或声音系统；
4. 不使用 D20，只使用两枚 D6；
5. 不把整张原型图当作网页背景；
6. 不自行生成或寻找替代视觉素材；
7. 缺少正式资产时使用清晰命名的占位块，并保留最终资源路径；
8. 所有动态文本必须由 HTML 渲染。

## 页面状态

实现单页状态机：

```ts
type AppState = 'idle' | 'rolling' | 'result';
```

### idle

- 左上角双骰图标；
- `关于检定`、`检定记录`；
- 标题 `上帝掷骰子吗？`；
- 副文案；
- 输入框；
- `掷骰检定`；
- 骰子免责声明；
- 动态技能低语。

### rolling

- 显示 `你的脑内正在请求一次检定……`；
- 输入框锁定；
- 两枚骰子执行 700–1000ms 的不同轨迹动画；
- 页面亮度略降；
- 技能名闪现；
- 动画结束后进入结果态。

### result

- 显示两枚骰子和总点数；
- 按规则生成结果等级；
- 输出 2–4 条技能发言；
- 输出最终判定；
- 提供 `再检定一次`、`换一批技能发言`、`生成分享图`；
- 保存到本地检定记录。

## 2D6 判定

- 双 1：大失败；
- 3–5：失败；
- 6–8：混沌成功；
- 9–11：成功；
- 双 6：大成功。

## 资产接入

所有资源路径必须按照 `docs/05-development-handoff.md` 预留。

若资源文件暂时不存在：

- 不报构建错误；
- 使用 CSS 占位；
- 在代码中保留最终资源路径常量；
- 添加 `TODO(asset)` 注释；
- 不使用随机网络图片替代。

## 验收标准

- 桌面端和移动端均可完成一次完整检定；
- 同一问题连续检定五次不完全重复；
- 非问句和荒诞输入可得到结果；
- 页面无登录入口、社区入口、思维图谱入口和声音入口；
- 左上角为双骰图标；
- `你的脑内正在请求一次检定……` 只出现在 rolling 状态；
- 所有可变内容可复制、可访问，不是图片文字；
- 资产替换不需要修改业务组件逻辑；
- 构建通过，无 TypeScript 错误。
