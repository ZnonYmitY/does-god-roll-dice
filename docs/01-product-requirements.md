# 上帝掷骰子吗？｜产品需求文档 PRD v1.0

## 1. 产品定义

### 1.1 产品名称

**上帝掷骰子吗？**

名称同时指向：

- 两枚六面骰构成的实际交互；
- “上帝不掷骰子”所代表的确定性与随机性争论；
- 用宇宙尺度的问题包装“今晚要不要吃炸鸡”一类微小决定的荒诞反差。

产品不是量子力学科普工具。量子随机性仅作为品牌隐喻和少量界面语言，不出现大面积公式、星空、原子轨道或科技 HUD。

### 1.2 产品定位

一款致敬《极乐迪斯科》技能检定体验的轻量互动网页。

用户输入一个问题、陈述、念头或情绪碎片，系统掷出 2D6，根据点数判定结果等级，召唤若干脑内技能发表互相矛盾的意见，最后输出一条带有黑色幽默、存在主义和失败美学的判定。

### 1.3 核心体验

> 把一件不值得惊动上帝的小事，包装成一次郑重其事的精神检定。

产品重点不是“替用户作出正确决定”，而是：

1. 把纠结外化为脑内技能争吵；
2. 提供随机性带来的仪式感；
3. 生成值得重复游玩和分享的荒诞文本；
4. 在严肃视觉与琐碎问题之间制造反差。

---

## 2. v1.0 范围

### 2.1 v1.0 必须实现

- 任意文本输入，不限制“要不要”句式；
- 关键词分类和关键对象提取；
- 两枚六面骰的随机结果和滚动动效；
- 五档结果等级；
- 从技能池中抽取 2–4 个技能发言；
- 输出最终判定；
- 同一个问题多次检定时降低重复率；
- 已读乱回分类；
- 重新检定；
- 保留骰点、仅更换技能发言；
- 生成和保存分享图；
- 桌面端与移动端响应式适配。

### 2.2 v1.0 明确不做

- 大模型生成或润色；
- 小模型蒸馏或微调；
- 难度等级；
- 技能值；
- 修正值；
- 用户状态问卷；
- 多轮追问；
- 登录、账户和云端历史；
- 严肃医疗、法律、投资或安全建议。

### 2.3 为什么移除难度与修正

难度、技能值和修正值可以增强规则还原度，但会在 v1.0 带来三类负担：

- 需要对任意问题判断难度和上下文；
- 需要额外文案解释数值来源；
- 容易把娱乐工具误导为理性决策系统。

v1.0 只保留 2D6 和结果档位，优先验证“输入—掷骰—技能发言—判定—分享”是否有趣。

## 3. 首页信息架构

### 3.1 固定文案

**产品名**

> 上帝掷骰子吗？

**主标题**

> 你的脑内正在请求一次检定。

**副文案**

> 某些问题本不值得回答。  
> 遗憾的是，它们已经醒了。

**按钮**

> 掷骰检定

**按钮下方说明**

> 两枚骰子对此表示愿意负责。  
> 但它们不会解释结果。

### 3.2 输入框

输入框允许：

- 标准疑问句：今晚要不要吃炸鸡？
- 简略问句：发不发消息？
- 非问句：不想上班。
- 单一对象：奶茶。
- 情绪碎片：我好累。
- 荒诞输入：猫是不是在审判我？
- 意识流输入：宇宙为什么还不下班？

随机 placeholder 示例：

- 今晚要不要吃炸鸡？
- 我不想上班。
- 猫是不是在审判我？
- 世界已经这样了，我还要健身吗？
- 给他发消息是勇敢，还是二次伤害？
- 这件事值得我从床上起来吗？

### 3.3 首页动态低语

页面空闲时，每 8–12 秒轮换一组两句技能对话，用于说明玩法：

> **LOGIC**：这个页面无法替你作出可靠决定。  
> **DRAMA**：那么它已经比大多数决定工具诚实了。

或：

> **INLAND EMPIRE**：不要输入。它们会知道的。  
> **ELECTROCHEMISTRY**：输入炸鸡。

## 4. 核心用户路径

```text
进入首页
→ 输入问题或精神噪声
→ 点击“掷骰检定”
→ 系统分类并提取对象
→ 两枚 D6 滚动
→ 加载文案随机出现
→ 显示点数和结果等级
→ 2–4 个技能依次发言
→ 显示最终判定
→ 再检定 / 换一批发言 / 生成分享图 / 修改问题
```

### 4.1 示例

输入：

> 今晚炸鸡。

识别结果：

```text
分类：food
对象：炸鸡
主技能候选：Electrochemistry
反对技能候选：Logic、Volition
氛围技能候选：Shivers、Inland Empire
```

骰点：

```text
5 + 3 = 8
混沌成功
```

技能发言：

> **ELECTROCHEMISTRY / 电化学**  
> 炸鸡不是答案。炸鸡是答案还没学会说话之前的样子。

> **LOGIC / 逻辑思维**  
> 你已经知道这不理性。你只是希望骰子替你承担责任。

> **SHIVERS / 寒颤**  
> 城市的夜风穿过外卖骑手的车筐。一种金黄色的命运正在抵达楼下。

最终判定：

> **【ELECTROCHEMISTRY：混沌成功】**  
> 吃吧。但别说这是胜利。今晚这更像一次停火。

## 5. 输入识别

### 5.1 分类层级

1. **关键词命中**：识别食物、社交、工作、消费等类别；
2. **对象提取**：提取“炸鸡”“朋友圈”“上班”等可进入模板的词；
3. **权重冲突处理**：多个分类命中时，选强关键词得分最高者；
4. **荒诞识别**：命中超现实、宇宙、动物审判等词时可进入 absurd；
5. **兜底**：无法稳定识别时进入 unknown，并使用“已读乱回”模板。

### 5.2 分类

| ID | 分类 | 示例 |
|---|---|---|
| `food` | 食物欲望 | 炸鸡、奶茶、火锅、咖啡 |
| `social` | 社交关系 | 聚会、朋友、发消息、前任 |
| `work_study` | 学习工作 | 上班、学习、DDL、写文档 |
| `shopping` | 消费冲动 | 买、下单、耳机、裙子 |
| `rest` | 休息摆烂 | 睡觉、躺、摆烂、不想动 |
| `creation` | 创作行动 | 写作、画图、做网站、剪视频 |
| `life` | 人生混沌 | 辞职、未来、意义、选择 |
| `absurd` | 荒诞乱回 | 猫审判、宇宙下班、天花板说话 |
| `unknown` | 无法识别 | 极短文本、随机字符、无关键词 |

### 5.3 已读乱回

无法理解不意味着报错。系统应表现为“似乎读懂了，但读懂的方向明显不太正常”。

例：

> 输入：猫会不会审判我？

> **INLAND EMPIRE**：它已经审判过了。只是还没有宣布判决。

> **LOGIC**：从法律意义上说，猫没有审判权。从客厅实际权力结构来看，这一点仍有争议。

> **SHIVERS**：窗台上，一只小型君主正在舔爪子。城市保持沉默。

> **判定**：会。但不是今天。今天它只是在记录证据。

## 6. 技能系统

v1.0 复用《极乐迪斯科》24 个技能名称和基础人格设定，用于个人非商业 Demo。

### 6.1 Intellect / 智力

- Logic / 逻辑思维
- Encyclopedia / 博学多闻
- Rhetoric / 修辞
- Drama / 戏剧
- Conceptualization / 概念化
- Visual Calculus / 视觉演算

### 6.2 Psyche / 精神

- Volition / 意志力
- Inland Empire / 内陆帝国
- Empathy / 共情
- Authority / 权威
- Esprit de Corps / 团队精神
- Suggestion / 暗示

### 6.3 Physique / 体格

- Endurance / 坚忍
- Pain Threshold / 痛觉阈值
- Physical Instrument / 肉体工具
- Electrochemistry / 电化学
- Shivers / 寒颤
- Half Light / 半明

### 6.4 Motorics / 身手

- Hand/Eye Coordination / 手眼协调
- Perception / 感知
- Reaction Speed / 反应速度
- Savoir Faire / 花样手腕
- Interfacing / 接口能力
- Composure / 镇定

### 6.5 技能抽取规则

每类问题配置：

- 1–3 个主技能候选；
- 1–3 个反对技能候选；
- 2–4 个氛围技能候选。

单次输出抽取：

- 1 个主技能；
- 1 个反对技能；
- 0–2 个氛围技能。

结果倾向：

- 大失败 / 失败：反对技能、Half Light、Inland Empire 等更容易主导；
- 混沌成功：主技能与混乱技能均可主导；
- 成功 / 大成功：主技能更容易主导。

---

## 7. 2D6 结果规则

| 结果 | 条件 | 文案方向 |
|---|---|---|
| 大失败 | 双 1 | 灾难、印刷错位、命运公开嘲笑 |
| 失败 | 总点数 3–5 | 撤退、拒绝、保护性失败 |
| 混沌成功 | 总点数 6–8 | 可以做，但附带奇怪代价 |
| 成功 | 总点数 9–11 | 行动获得支持，但不做鸡汤 |
| 大成功 | 双 6 | 命运短暂开绿灯，仍保持荒诞 |

结果页只展示：

```text
[ELECTROCHEMISTRY CHECK]
5 + 3 = 8
混沌成功
```

不展示难度、技能值、修正值或总目标值。

## 8. 掷骰交互

### 8.1 骰子

- 两枚六面骰；
- 一枚象牙灰、黑色点数；
- 一枚暗酒红、米白点数；
- 表面磨损、边角不规则；
- 是旧桌游盒里偶然凑到一起的物件，不是魔法神器。

### 8.2 动效阶段

1. 页面亮度下降，输入区失焦；
2. 显示“当前状态尚未决定”；
3. 两枚骰子从右下桌面滚入，旋转速度不同；
4. 出现短暂油彩拖影和技能名闪现；
5. 骰子碰撞并停下；
6. `REACTION SPEED` 先说：“它们停下了。”；
7. 显示骰点和结果；
8. 显示“观测完成。解释权已移交脑内。”；
9. 技能发言逐条进入。

建议总动效时长 1.8–2.8 秒，可跳过，但不应超过 3 秒。

---

## 9. 结果页

### 9.1 信息结构

1. 主检定技能名称；
2. 两枚骰子的点数和总值；
3. 结果等级；
4. 2–4 条技能发言；
5. 最终主导技能与判定；
6. 操作按钮。

### 9.2 操作

- **再检定一次**：重新掷骰并生成全部结果；
- **换一批发言**：保留问题、骰点、结果等级和主技能，只替换技能发言与最终措辞；
- **生成分享图**：打开分享图预览；
- **修改问题**：回到输入区并保留原文本。

## 10. 分享图

分享图是 v1.0 P0 功能。

### 10.1 尺寸

- 竖版：1080 × 1440，优先实现；
- 方版：1080 × 1080，可作为 P1。

### 10.2 内容

- 产品名；
- 观测编号；
- 用户输入；
- 主技能与肖像；
- 2D6 结果；
- 结果等级；
- 1–2 条精选技能发言；
- 最终判定；
- 固定或随机页脚。

### 10.3 页脚文案

固定推荐：

> 两枚骰子对此表示愿意负责。  
> 但它们不会解释结果。

随机替代：

- 上帝尚未回应。骰子已经回应。
- 观测完成。责任归属不明。
- 命运已读。解释失败。
- 如果你不同意，可以再掷一次。

### 10.4 三种版式

1. **技能档案版**：技能肖像最大，突出主导技能；
2. **脑内记录版**：突出多条技能对话；
3. **城市旁白版**：突出 Shivers / Inland Empire 式旁白和城市背景。

### 10.5 生成方案

```text
结果数据
→ 渲染隐藏分享卡 DOM
→ html2canvas / satori 生成图片
→ 预览
→ 保存 PNG
```

所有生成均在浏览器本地完成，不上传用户输入。

## 11. 历史去重

localStorage 记录最近 10 次：

- 输入标准化文本；
- 分类；
- 骰点；
- 技能组合；
- 技能文案 ID；
- 最终判定 ID。

规则：

- 最近 3 次不重复最终判定；
- 最近 5 次不重复完全相同技能组合；
- 同一输入连续检定时优先未出现文案；
- 池不足时允许重复，但更换主导技能或辅助技能。

---

## 12. 安全与边界

对医疗、法律、投资、自伤、安全等明显高风险内容，系统不输出肯定或否定行动建议，改用统一提醒：

> **VOLITION**：这件事不该交给两枚骰子。  
> 找一个现实中能够承担责任的人，和你一起处理它。

页面底部常驻：

> 本工具仅供娱乐。骰子不会承担现实责任。

---

## 13. 验收标准

### 功能

- 任意正常文本均能产生结果；
- 非问句与荒诞输入不会报错；
- 骰子始终为两枚 D6；
- 每次显示 2–4 条技能发言；
- 同一输入连续 5 次不完全重复；
- 分享图可预览和保存；
- 移动端无横向滚动和遮挡。

### 内容

- 24 个技能均有名称、属性、语气和适用分类；
- 至少 9 个问题分类；
- MVP 文案量不少于 350 条；
- 推荐上线文案量 600–800 条；
- 已读乱回拥有独立加载文案、技能发言和判定池。

### 性能

- 首屏可交互时间控制在 2 秒左右；
- 单次检定总等待不超过 3 秒；
- 原作肖像按需加载，不阻塞首页；
- 分享图生成失败时提供原生截图提示。
