# 上帝掷骰子吗？｜视觉与交互设计方案 v1.0

## 1. 设计方向

### 1.1 核心公式

> **破败都市表现主义油画 × 抽象技能肖像 × 旧印刷界面 × 两枚磨损 D6 × 微量“精神测量”语言**

不是：

- 中世纪奇幻；
- 魔法阵和水晶球；
- 紫色魔法光；
- 羊皮卷和金色雕花框；
- D20；
- 量子科普网站；
- 蓝紫科技 HUD。

### 1.2 视觉叙事

> 世界很安静，脑内很吵。

环境层低饱和、潮湿、破败；技能发言时，高饱和抽象颜色侵入页面。首页静止时偏城市与房间，检定开始后才逐渐出现技能肖像和鲜艳笔触。

### 1.3 自有特色

借“量子测量”作为非常轻的隐喻：

- 掷骰前：状态未决；
- 骰子落下：观测完成；
- 结果出现：状态坍缩；
- 技能发言：解释权移交脑内。

建议视觉占比：

- 65% 破败城市油画和技能意识流；
- 25% 两枚骰子与精神测量概念；
- 10% 观测编号、刻度和系统小字。

## 2. 首页布局

### 2.1 桌面端

采用左右不对称布局，不使用普通 AI 工具的居中卡片结构。

```text
┌────────────────────────────────────────────────────────┐
│ 上帝掷骰子吗？                          声音 / 关于      │
│                                                        │
│ 你的脑内正在请求一次检定。      右侧：潮湿城市与房间油画 │
│ 某些问题本不值得回答。          桌面上放着两枚旧 D6     │
│ 遗憾的是，它们已经醒了。        技能肖像藏在暗处         │
│                                                        │
│ ┌──────────────────────────────────────────────┐       │
│ │ 把问题、念头或任何精神噪声放在这里……          │       │
│ └──────────────────────────────────────────────┘       │
│                                      [ 掷骰检定 ]       │
│ 两枚骰子对此表示愿意负责。                              │
│ 但它们不会解释结果。                                    │
│                                                        │
│ LOGIC：这个页面无法替你作出可靠决定。                    │
│ DRAMA：那么它已经比大多数决定工具诚实了。                │
└────────────────────────────────────────────────────────┘
```

左侧约 56%，右侧约 44%。左侧需要大面积低细节留白；右侧承载城市、桌面、骰子和隐藏的技能肖像。

### 2.2 移动端

- 顶部 Logo；
- 主标题和副文案；
- 输入区；
- 骰子与按钮；
- 小字说明；
- 底部技能低语；
- 背景场景裁切到右上或底部，不与正文竞争。

移动端输入后，结果在同一页面纵向展开，不切换到独立路由。

## 3. 背景

### 3.1 场景建议

- 凌晨的旧房间；
- 窗外潮湿街道和模糊路灯；
- 桌上一杯冷咖啡、旧电话、撕裂的纸张；
- 两枚 D6 放在桌面边缘；
- 墙面剥落、海报被撕去一半；
- 房间边缘潜藏不完整的技能面孔。

背景不画明确主角，不做完整叙事插画。它应该像一段没有加载完整的记忆。

### 3.2 构图

- 左上：煤灰色、细节较少，用于 Logo 和标题；
- 左下：旧纸、墙体纹理；
- 右上：窗户、雾气、城市灯光；
- 右下：桌面、骰子、浓重笔触；
- 四周：颜料与黑暗不规则侵入。

### 3.3 色板

开发参考值：

| 用途 | 色值 |
|---|---|
| 煤灰黑 | `#171918` |
| 湿墙灰绿 | `#313B36` |
| 石油蓝 | `#24353C` |
| 烟灰白 | `#D5D0C4` |
| 褪色路标黄 | `#B49A4C` |
| 锈红 | `#873F32` |
| 暗酒红 | `#5C2B32` |

实际画面不得呈现为干净扁平色块，需要叠加油画、纸张、颗粒和印刷纹理。

## 4. 字体与排版

### 4.1 标题

- 中文标题使用有笔锋、略显狭长或旧印刷感的宋体/衬线体；
- 不使用仙侠书法字体；
- 英文技能名使用窄体、全大写、较高字距；
- Logo 可使用两行结构，但保持自然断句：

```text
上帝掷骰子吗？
DOES GOD ROLL DICE?
```

英文只作微型辅助，不喧宾夺主。

### 4.2 正文

- 中文正文使用可读性较好的宋体或中性黑体；
- 技能发言行距偏宽；
- 避免所有文字装在卡片中；
- 技能名、正文和系统小字形成三级信息层级。

### 4.3 排版特征

- 少量错位；
- 局部印刷重影；
- 不完整分割线；
- 大片留白；
- 技能发言可打破网格，但正文阅读顺序必须清楚。

---

## 5. 输入组件

### 5.1 外观

- 半透明或接近透明；
- 无大圆角；
- 1px 不完整边框；
- 左上边框有缺口；
- 聚焦时出现颜料渗透，而非霓虹发光；
- 粗竖线光标；
- 右下角显示 `INPUT 023 / 120`。

### 5.2 标签随机池

- CURRENT MATTER
- UNRESOLVED ISSUE
- SPIRITUAL NOISE
- 待处理问题
- 一件不值得惊动上帝的事

### 5.3 状态

- 默认：低对比；
- Hover：边框局部浮现；
- Focus：背景出现缓慢移动的属性色块；
- 输入完成：右下显示“状态未决”；
- 检定中：输入框失焦并降低透明度；
- 结果后：缩小为顶部问题摘要，可点击修改。

## 6. 按钮

### 6.1 主按钮

文案：**掷骰检定**

外形像旧印刷标签或被盖章的文件按钮，不使用金属雕花、游戏宝箱风或发光水晶。

状态：

- 默认：烟灰白文字，旧纸或暗色底；
- Hover：锈红笔触从边缘侵入，旁边浮出“它们听见了。”；
- Pressed：文字短暂错位；
- Loading：变为“正在检定……”并淡出，为骰子让位；
- Disabled：低饱和，提示“先放进一点精神噪声”。

### 6.2 次级按钮

- 再检定一次；
- 换一批发言；
- 生成分享图；
- 修改问题。

次级按钮以文字和不完整下划线为主，避免四个大型按钮并排。

## 7. 两枚六面骰

### 7.1 外观

**骰子 A：象牙灰**

- 灰白旧塑料；
- 黑色点数；
- 边角磨圆；
- 有细小划痕和污点。

**骰子 B：暗酒红**

- 深酒红旧塑料；
- 米白点数；
- 略有半透明感；
- 内部有杂质，不像宝石。

两枚骰子款式不完全一致，像从不同盒子里凑来的物件。

### 7.2 待机状态

- 真实放在桌上，不悬浮；
- 一枚平躺，一枚压住纸角；
- Hover 只发生 1–2px 位移或阴影变化；
- 可显示小字：`2D6 / TWO UNCERTIFIED MEASUREMENT TOOLS`。

### 7.3 动效

**准备阶段**

- 背景降低 10% 亮度；
- 输入框失焦；
- 骰子出现 2–3 层重影；
- 显示“当前状态尚未决定”。

**滚动阶段**

- 两枚骰子从右下滚到页面中心；
- 动画 700–1000ms；
- 两枚骰子旋转速度和轨迹不同；
- 有轻微真实碰撞；
- 留下短暂油画拖影；
- 背景闪现 LOGIC、VOLITION、ELECTROCHEMISTRY 等技能名。

**停止阶段**

> **REACTION SPEED**  
> 它们停下了。

随后展示点数。

**结果阶段**

> `4 + 3 = 7`  
> `混沌成功`

系统小字：

> 观测完成。解释权已移交脑内。

### 7.4 结果差异

- 大失败：页面轻微错位、暗红刮痕、一枚骰子偏离中心；
- 失败：降低饱和度，停顿略长；
- 混沌成功：两种属性色彩同时侵入；
- 成功：主技能色彩占主导；
- 大成功：不是金光，而是一笔突然明亮的油彩。

## 8. 技能肖像

### 8.1 v1.0 方案

首版直接使用《极乐迪斯科》原作技能肖像作为个人 Demo 占位素材，避免将大量时间投入 24 张重绘。

使用原则：

- 不修改原图本身；
- 不把原图打包为可再分发的素材库；
- 仓库中仅保留路径和命名说明，素材由开发者本地添加；
- 页面明确标注非官方粉丝致敬；
- 公开部署前再次评估版权风险。

### 8.2 三种展示状态

**沉睡**

- 肖像 8%–18% 可见度；
- 藏在背景油画里；
- 不出现完整边框。

**发言**

- 对应肖像局部浮出；
- 饱和度上升；
- 技能名先出现，正文随后打字；
- 肖像可只露出脸、手、纹理或颜色。

**主导**

- 最终判定时占结果区约 28%–36%；
- 属性色彩短暂侵入整个页面；
- 分享图将该肖像作为主视觉。

### 8.3 属性颜色

- Intellect：冷蓝、灰青；
- Psyche：紫红、洋红；
- Physique：橙红、锈黄；
- Motorics：青绿、灰绿。

页面不画整齐四色条，颜色应像颜料从肖像里溢出。

## 9. 技能发言

### 9.1 不使用聊天气泡

结构：

```text
ELECTROCHEMISTRY
炸鸡不是答案。炸鸡是答案还没学会说话之前的样子。
```

- 技能名全大写；
- 左侧或背景带技能肖像；
- 发言直接排版在页面；
- 不使用左右对话气泡；
- 不用完整卡片包住每句话。

### 9.2 进入动画

1. 肖像色彩先在背景中扩散；
2. 技能名以短促闪现进入；
3. 正文按语义停顿逐段出现；
4. 下一条发言覆盖或挤压上一条的视觉空间；
5. 最终判定前留 300–500ms 空白。

### 9.3 音效

可选：

- 骰子滚动和桌面碰撞；
- 纸张摩擦；
- 很轻的打字声；
- 低频城市环境声；
- 大失败时短促音频失真。

默认静音，用户主动打开。

## 10. 结果页

结果不是单独的中央卡片，而是首页状态转化：

- 输入区向上收起；
- 骰子停在页面中下部；
- 技能发言占据左侧或主体；
- 主导技能肖像从背景浮出；
- 最终判定在底部形成最大文字块；
- 操作以小型文字按钮收尾。

结果标题：

```text
[ELECTROCHEMISTRY CHECK]
5 + 3 = 8
混沌成功
```

最终判定：

```text
ELECTROCHEMISTRY：混沌成功
吃吧。但别说这是胜利。今晚这更像一次停火。
```

## 11. 分享图

### 11.1 技能档案版

- 左上：产品名与观测编号；
- 左侧：用户问题、骰点和结果；
- 右侧或底部：主技能肖像；
- 下方：最终判定；
- 页脚：骰子免责声明。

### 11.2 脑内记录版

- 2–3 个技能发言构成主体；
- 肖像以小幅裁切或背景色块出现；
- 像一份不完整的审讯记录或精神档案。

### 11.3 城市旁白版

- 使用背景城市油画；
- 只显示一条 Shivers / Inland Empire 式旁白；
- 最终判定大字号；
- 适合失败、混沌成功和人生类输入。

### 11.4 分享图示例

```text
上帝掷骰子吗？
OBSERVATION #002731

用户提交：
世界已经这样了，我还要健身吗？

PHYSICAL INSTRUMENT CHECK
5 + 4 = 9
成功

PHYSICAL INSTRUMENT：
站起来。哪怕只是为了证明沙发尚未取得永久统治权。

最终判定：
去。今天不要求胜利，只要求身体重新参与现实。

两枚骰子对此表示愿意负责。
但它们不会解释结果。
```

## 12. 组件与资产清单

### 12.1 必须设计

- Logo 字标；
- 桌面首页背景；
- 移动端首页背景裁切；
- 两枚 D6 的 1–6 点数状态；
- 输入框 5 种状态；
- 主按钮 5 种状态；
- 技能发言样式；
- 骰点结果组件；
- 四个结果操作；
- 分享图三套版式；
- 纹理：油画、旧纸、颗粒、印刷错位。

### 12.2 动效

- 骰子待机；
- 骰子滚动与碰撞；
- 油彩拖影；
- 技能名闪现；
- 肖像沉睡到发言；
- 文本打字；
- 大失败错位；
- 大成功亮色笔触；
- 分享图预览进入。

### 12.3 首批原作肖像

优先接入：

1. Logic
2. Volition
3. Inland Empire
4. Electrochemistry
5. Shivers
6. Drama
7. Composure
8. Conceptualization

其余技能可先使用属性底色和文字，后续逐步补齐。

## 13. 设计验收

- 第一眼不能被识别为中世纪或魔法 RPG；
- 页面必须清楚出现两枚 D6；
- 首页不操作时也能理解“脑内技能会发言”；
- 技能肖像与页面背景融合，不是普通头像卡；
- 量子隐喻不能压过游戏体验；
- 分享图在没有产品上下文时仍能独立成立；
- 所有动画在 `prefers-reduced-motion` 下可降级；
- 移动端技能发言不与肖像互相遮挡。
